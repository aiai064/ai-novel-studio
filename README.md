# AI 小说工作台

一个用于长篇小说创作的本地 MVP。它把“AI 写小说”拆成项目设定、大纲、章节生成、一致性检查、人物状态、伏笔表和长期记忆几个模块，先跑通创作闭环。

## 运行

```bash
npm install
npm run dev
```

当前开发服务已启动在：

```text
http://127.0.0.1:5174
```

## DeepSeek 配置

在左侧“DeepSeek 接口”里填写：

- Base URL：`https://api.deepseek.com`
- 模型名：`deepseek-v4-flash`
- API Key：你的 DeepSeek Key
- 上下文：默认 `全文上下文`，会把已生成章节正文纳入后续生成；可切换为 `智能摘要` 以节省输入。
- 请求缓存：默认开启，完全相同的 DeepSeek 请求会命中本地代理缓存，避免重复消耗。

应用会通过本地 `/api/deepseek/chat` 代理请求 DeepSeek，避免浏览器跨域问题。当前 Key 仍会保存在浏览器 `localStorage`，正式上线时应改为服务端环境变量或用户级密钥托管。

## 已实现

- 小说项目种子：书名、类型、核心创意、章节数、目标字数、文风。
- AI 生成项目：logline、人物卡、世界观条目、伏笔表、章节目标和场景纲。
- 章节流水线：选择章节、编辑目标/场景纲、生成正文草稿。
- 上下文预览：查看当前章节发送给 DeepSeek 的项目、记忆、设定、伏笔和前文摘要。
- 全文上下文：利用 DeepSeek V4 的 1M context，在生成章节时可携带已生成章节正文。
- 请求缓存：本地代理缓存 24 小时内的相同请求，并保持稳定上下文前缀以提高模型侧缓存命中。
- 记忆闭环：章节生成后自动更新章节摘要、全局记忆、人物状态、伏笔状态。
- AI 记忆整理：压缩长期记忆，刷新人物状态、世界规则和伏笔表。
- 设定库编辑：可手动修订人物、世界规则、伏笔状态和回收章节。
- 事件时间线：章节生成和记忆整理会沉淀 timeline events，辅助检查时间、地点、人物状态连续性。
- 章节后处理：章节生成 JSON 可返回人物更新、世界规则更新、伏笔更新、时间线事件，自动写回项目。
- 长章节生成：按“目标总字数 / 章节数”计算每章目标，范围 3000-6000 字；不足 90% 时自动续写补足，最多两轮。
- DeepSeek 接入：生成项目圣经、优化章节流水线，并根据章节目标、前文摘要、设定库、伏笔表生成正文。
- 一致性检查：基础检查章节长度、主角出现、结尾钩子、逾期伏笔。
- 本地持久化：使用 `localStorage` 保存项目和模型设置。
- 导入导出：支持 Markdown 导出、JSON 项目备份和 JSON 恢复。

## 参考方向

- Novel-OS：三层上下文思想，确保模型拿到标准、小说设定和正文资料。
- Errata：Librarian 记忆工具、rolling continuity、contradiction tracking、通用导入导出。
- Novelgen：先生成章节概要，再逐章执行，长篇大纲按章节拆解。
- AI_NovelGenerator：围绕长篇一致性、上下文和多章节生成流程设计。

## 后续路线

1. 后端化：增加 FastAPI/NestJS 服务，避免浏览器直接保存 API Key。
2. 数据库：PostgreSQL + pgvector，保存章节、设定、长期记忆和向量检索结果。
3. 真模型增强：继续完善 DeepSeek 生成参数、流式输出、失败重试和成本统计。
4. 高级检查：人物关系、时间线、能力规则、伏笔回收、视角稳定性。
5. 批量生产：任务队列、章节批量生成、版本对比、成本统计。
